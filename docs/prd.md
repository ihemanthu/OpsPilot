# Product Vision

OpsPilot AI Lite enables users to submit operational or support requests through voice or chat, automatically converts them into structured tickets using AI, triggers workflow automation, and provides real-time visibility into the request lifecycle.

---

# Executive Summary

Most operational requests originate as conversations.

A user calls support, leaves a voice note, reports an issue verbally, or sends an unstructured message.

Someone must manually:

1. Listen to the request
2. Understand the problem
3. Create a ticket
4. Notify stakeholders
5. Track progress

OpsPilot AI automates this flow.

The system captures voice input, extracts structured information using AI, creates a ticket, triggers downstream automation, and displays results in a unified dashboard.

* Store transcript
* Generate structured output

---

## FR-004 Ticket Management

System shall:

* Create ticket
* Retrieve ticket
* Restrict access to owner

---

## FR-005 Workflow Automation

System shall:

* Trigger n8n workflow
* Store execution status
* Display results

---

### Retry Policy

If n8n webhook execution fails:

1. Retry immediately
2. Retry after 5 seconds
3. Retry after 15 seconds

Maximum retries: 3

If all retries fail:

- Mark workflow_run as FAILED
- Store error payload
- Surface failure in dashboard

---

## FR-006 Realtime Updates

System shall:

* Provide a WebSocket connection for dashboard updates
* Emit events for transcript, ticket, and workflow changes
* Enforce JWT validation and resource ownership on websocket connections

---

## FR-007 System Resilience & Observability

System shall:

* Support Idempotency-Key for write operations (e.g. voice session creation)
* Use X-Correlation-ID headers for end-to-end request tracing

---

## FR-008 Search & Pagination

System shall:

* Support cursor/page-based pagination for list endpoints
* Support filtering by status, priority, and searching by keyword

```sql
    avatar_url TEXT,
    provider VARCHAR(50),
```

# Table: tickets

```sql
CREATE TABLE tickets (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES users(id),

    ticket_number VARCHAR(50) UNIQUE NOT NULL,

    title TEXT NOT NULL,
    description TEXT NOT NULL,

    priority VARCHAR(20) NOT NULL,

    status VARCHAR(20) NOT NULL,

    source VARCHAR(20) NOT NULL,

    created_at TIMESTAMP DEFAULT NOW()
);
```

# API Specification

> **Note**: This section provides a high-level overview. For the strict REST contract, request envelopes, and complete schemas, refer to `api-contracts.md`.

## Base API Path

```text
/api/v1
```

## Core Namespaces

- **Authentication (`/api/v1/auth`)**: Google OAuth2 login, callback, logout, and current user info.
- **Voice (`/api/v1/voice`)**: Voice session creation and completion (Idempotent).
- **Tickets (`/api/v1/tickets`)**: Ticket creation, retrieval, and listing with pagination, search, and filtering.
- **Workflows (`/api/v1/workflow-runs`)**: Workflow status tracking and event logs.
- **WebSocket (`/api/v1/ws`)**: Real-time events for transcripts, tickets, and workflow updates.
- **Internal (`/api/v1/internal`)**: Protected callbacks for Pipecat extraction and n8n webhook status.
- **Health (`/api/v1/health`)**: Liveness and readiness probes.

# Error Handling

All API errors adhere to a standard JSON envelope with machine-readable codes and human-readable messages, as specified in `api-contracts.md`.

## Standard Error Envelope

```json
{
  "success": false,
  "correlation_id": "uuid",
  "error": {
    "code": "ERROR_CODE",
    "message": "Human readable message"
  }
}
```

## Common Error Codes

- **Authentication**: `UNAUTHORIZED`, `SESSION_EXPIRED`, `OAUTH_FAILED`
- **Voice**: `VOICE_SESSION_NOT_FOUND`, `VOICE_SESSION_TIMEOUT`, `TRANSCRIPT_EMPTY`
- **Tickets**: `TICKET_NOT_FOUND`, `INVALID_TICKET_DATA`, `VALIDATION_ERROR`
- **Workflows**: `WORKFLOW_FAILED`, `WORKFLOW_RUN_NOT_FOUND`
- **Infrastructure**: `INTERNAL_SERVER_ERROR`, `RATE_LIMIT_EXCEEDED`
